# L3-M1-S04 压测计划与验证指标

## 一句话结论

- 压测计划与验证指标 是 L3 阶段的关键能力点，面试回答建议覆盖“定义、原理、场景、边界”。

## 结构图

```mermaid
flowchart LR
    A[业务目标] --> B[架构分层]
    B --> C[容量与稳定性设计]
    C --> D[演进治理]
```

## 核心知识点

1. 架构设计先回答瓶颈在哪里，再谈技术名词。
2. 容量评估、压测、降级预案要同时存在。
3. 演进方案需要兼顾收益、复杂度和交付节奏。

## 高频面试题

### Q1：你如何在项目中落地“压测计划与验证指标”？

答题骨架：
1. 先说明业务目标和约束。
2. 再给可执行方案和关键指标。
3. 最后补充风险、边界与回退策略。

### Q2：压测计划与验证指标 的常见误区是什么？

答题骨架：
1. 说明常见错误做法。
2. 给出正确实践和适用条件。
3. 用一个真实场景收尾。

## Java 示例代码（含注释）

```java
public class CapacitySnippet {
    public static void main(String[] args) {
        long peakQps = 5000;
        int singleNodeQps = 1200;
        double redundancy = 1.3;
        // 容量估算 = 峰值 / 单机能力 * 冗余系数
        int nodes = (int) Math.ceil((peakQps / (double) singleNodeQps) * redundancy);
        System.out.println("nodes=" + nodes);
    }
}
```
