# L1-M3-S02 参数校验与统一异常

## 一句话结论

- 参数校验与统一异常 是 L1 阶段的关键能力点，面试回答建议覆盖“定义、原理、场景、边界”。

## 结构图

```mermaid
flowchart LR
    A[请求进入] --> B[分层处理]

    C --> D[结果输出]
```

## 核心知识点

1. 分层边界清晰是可维护性的第一前提。
2. 统一校验、异常、日志规范能显著降低问题定位成本。
3. 优先“简单可控”实现，再逐步演进到复杂治理。

## 高频面试题

答题骨架：
1. 先说明业务目标和约束。
2. 再给可执行方案和关键指标。

### Q2：参数校验与统一异常 的常见误区是什么？

答题骨架：
1. 说明常见错误做法。
2. 给出正确实践和适用条件。

## Java 示例代码（含注释）

```java
public class ExceptionSnippet {
    public static void main(String[] args) {
        try {
            // 参数校验失败时抛出业务异常
            createOrder(0);
        } catch (IllegalArgumentException e) {
            // 统一捕获后可映射成标准错误码
            System.out.println("biz-error=" + e.getMessage());
        }
    }

    static void createOrder(int count) {
        if (count <= 0) throw new IllegalArgumentException("count must > 0");
    }
}
```
