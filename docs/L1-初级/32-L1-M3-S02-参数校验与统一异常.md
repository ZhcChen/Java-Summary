## 前置知识

- 会写方法参数。
- 知道异常捕获基础语法。
- 理解输入数据可能不可信。

## 术语解释（零基础友好）

- **参数校验**：在进入业务逻辑前验证输入是否合法。
- **统一异常处理**：集中处理异常并输出标准响应。
- **错误码**：可被程序识别的稳定错误标识。

## 详细学习步骤（从不会到会）

1. 先定义输入对象及校验规则。
2. 在入口统一拦截校验失败并返回标准错误。
3. 对业务异常与系统异常分开处理。
4. 补充日志上下文，便于快速定位。

## 常见错误与纠偏

- 每个接口单独写一套异常返回，样式不一致。
- 只返回“失败”而不返回可定位信息。

## 练习任务（建议动手）

1. 设计一个创建用户请求对象并加校验规则。
2. 实现一个统一错误响应结构：code/message/data。

## 练习参考方向

- 校验失败应在业务逻辑前返回，减少无效处理。
- 错误码命名建议稳定、可检索。

## Java 示例代码（含注释，可直接运行）


**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

**预期输出（示例）：**
```text
biz-error=count must > 0
```

```java
public class Main {
    public static void main(String[] args) {
        try {
            // 业务入参校验失败时抛出异常
            createOrder(0);
        } catch (IllegalArgumentException e) {
            // 统一捕获后可映射统一错误码
            System.out.println("biz-error=" + e.getMessage());
        }
    }

    static void createOrder(int count) {
        if (count <= 0) throw new IllegalArgumentException("count must > 0");
    }
}
```
