## 核心知识点

1. 先看执行计划和扫描行数，再决定优化方向。
2. 索引设计要服务真实查询模式，不追求“索引越多越好”。
3. 优化后必须做基线对比，避免局部提速整体变慢。

## 前置知识

## 术语解释（零基础友好）

- **索引**：帮助数据库快速定位数据的数据结构。
- **事务**：把多条操作作为一个整体提交或回滚。
- **隔离级别**：控制事务之间相互可见性的规则。

## 详细学习步骤（从不会到会）

1. 先写无索引查询并观察扫描行数。
2. 给过滤字段加索引并比较执行计划。
3. 通过转账示例理解事务原子性。
4. 再比较不同隔离级别对一致性和性能的影响。

## 常见错误与纠偏

- 索引建得很多但查询条件不匹配，实际不生效。
- 事务范围过大，导致锁时间过长。
