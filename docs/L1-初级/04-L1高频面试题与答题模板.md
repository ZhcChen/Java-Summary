# L1-04 基础题表达模板与问答训练

> 本章不是让你“背八股”，而是训练你把已学知识稳定表达出来。

## 本章定位

- 你将学会：用固定结构讲清基础技术问题。
- 你将避免：回答散乱、只讲结论不讲原因。
- 你将产出：1 套可重复使用的答题模板 + 1 组自测清单。

## 前置知识

- 已学习 L1 的 01~03 模块。
- 能解释至少 5 个基础概念（集合、线程、索引、缓存等）。

## 一、统一答题结构（推荐 5 步）

1. **一句话结论**：先回答“是什么”。
2. **核心原理**：讲 2~3 个关键机制。
3. **业务场景**：说明在什么场景下使用。
4. **边界风险**：指出失效条件或常见坑。
5. **验证方式**：给出可观察结果或指标。

这个结构可用于学习复盘、组内分享、技术面试，通用性强。

## 二、L1 高频问题（训练清单）

| 编号 | 题目 | 一句话结论 |
|---|---|---|
| L1-Q01 | Java 为什么只有值传递？ | 参数传递的是值副本，引用类型传的是地址值副本 |
| L1-Q02 | `HashMap` 为什么线程不安全？ | 并发写入存在竞态，结构和数据都可能异常 |
| L1-Q03 | `synchronized` 与 `volatile` 区别？ | 前者互斥+可见性，后者仅可见性/有序性 |
| L1-Q04 | MySQL 索引为什么能提升查询效率？ | 通过有序结构减少扫描范围 |
| L1-Q05 | Redis 为什么常用于缓存？ | 内存访问快，能显著降低 DB 读压力 |

## 三、完整示例（同一题的低分和高分）

### 题目

`HashMap` 为什么线程不安全？

### 低分回答（示例）

“因为它没加锁，所以不安全。”

问题：只有结论，没有机制、场景和后果。

### 高分回答（示例）

- 结论：`HashMap` 在并发读写场景下不安全。
- 原理：`put` 过程不是原子操作，并发下可能发生数据覆盖与结构中间态可见。
- 场景：多线程共享本地缓存时风险高。
- 风险：数据丢失、读取异常、线上问题难复现。
- 处理：并发场景改用 `ConcurrentHashMap`，复合操作配合原子类或锁。

## 四、训练方法（可执行）

1. 每天选 3 题，每题控制在 90 秒内口述。
2. 录音回放，检查是否覆盖“五步结构”。
3. 对照本章评分表，低于 80 分的题第二天重答。

## 五、评分表（100 分）

| 维度 | 分值 | 说明 |
|---|---:|---|
| 结论清晰 | 20 | 第一段是否直接回答问题 |
| 原理准确 | 25 | 是否讲清关键机制 |
| 场景匹配 | 20 | 是否给出贴近业务的例子 |
| 风险边界 | 20 | 是否说明限制与失败情况 |
| 表达结构 | 15 | 条理是否清晰、时间是否可控 |

## 六、错答与修正（章级题解）

### 题目

为什么回答技术题要先讲“结论”，再讲“原理”？

### 错答示例（低分）

- “先讲什么都行，能说出来就可以。”

### 修正答法（高分）

1. 面试和技术沟通都是“有限时间高信息密度”场景。
2. 先给结论，能让对方快速建立上下文。
3. 再讲原理和边界，体现理解深度和工程判断。

### 打分差异

| 维度 | 低分答法 | 高分答法 |
|---|---|---|
| 沟通效率 | 没有主线 | 结论先行，信息聚焦 |
| 技术深度 | 无机制解释 | 有原理、有边界 |
| 可复用性 | 临场发挥 | 结构可复用 |

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
public class Main {
    public static void main(String[] args) {
        // 使用数组模拟“标准答题模板”五步结构
        String[] answerTemplate = {
            "1) 结论: HashMap 在并发写下不安全。",
            "2) 原理: put 过程不是原子操作，可能出现竞态。",
            "3) 场景: 多线程共享缓存时风险最大。",
            "4) 风险: 数据覆盖、结构异常、故障难复现。",
            "5) 验证: 并发压测观察错误率和数据一致性。"
        };

        // 按顺序输出，模拟一次结构化口述
        for (String line : answerTemplate) {
            System.out.println(line);
        }
    }
}
```

**预期输出示例：**

```text
1) 结论: HashMap 在并发写下不安全。
2) 原理: put 过程不是原子操作，可能出现竞态。
3) 场景: 多线程共享缓存时风险最大。
4) 风险: 数据覆盖、结构异常、故障难复现。
5) 验证: 并发压测观察错误率和数据一致性。
```
