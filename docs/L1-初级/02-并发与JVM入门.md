# L1-02 并发与 JVM 入门

> 本章目标：先建立“并发正确性”与“JVM 运行机制”的基本心智模型，再进入后续调优与排障。

## 本章定位

- 你将学会：线程协作、线程安全基础、JVM 内存区域认知。
- 你将避免：把 `volatile` 当锁、线程数无限加大、遇到 OOM 只会重启。
- 你将产出：1 个并发协作示例 + 1 份 JVM 区域说明卡。

## 前置知识

- 会写 Java 类与方法。
- 理解“多个任务可以并行执行”这件事。
- 能看懂基础日志输出。

## 学习路径（建议顺序）

1. 先理解线程生命周期：创建、运行、阻塞、结束。
2. 再理解线程安全三要素：可见性、原子性、有序性。
3. 最后理解 JVM 内存区域，知道常见错误对应哪个区域。

## 一、并发基础：不是“快”，先是“对”

### 1.1 `synchronized` 与 `volatile` 区别

- `synchronized`：提供互斥访问，同时保证可见性。
- `volatile`：保证可见性与有序性，不保证复合操作原子性。

### 1.2 典型误用

`count++` 在多线程下不是原子操作；即使 `count` 是 `volatile`，也可能丢数据。

### 1.3 实践建议

- 共享可变状态尽量少。
- 需要“读-改-写”时优先用锁或原子类。
- 不要先拍脑袋加线程，先看吞吐瓶颈在哪。

## 二、线程池基础：把并发变成可控系统

### 2.1 必懂参数

- `corePoolSize`：常驻线程数。
- `maximumPoolSize`：极限线程数。
- `workQueue`：任务排队位置。
- `RejectedExecutionHandler`：任务过载时的拒绝策略。

### 2.2 常见风险

- 队列无限大：看起来不拒绝，实际上延迟持续拉高。
- 线程过多：上下文切换成本高，CPU 忙但吞吐不上升。

## 三、JVM 内存区域入门

| 区域 | 存什么 | 常见问题 |
|---|---|---|
| 堆 | 对象实例 | 堆内存 OOM |
| 栈 | 方法调用栈帧 | 栈深过大导致栈溢出 |
| 方法区（元空间） | 类元信息 | 元空间不足 |
| 程序计数器 | 当前线程执行位置 | 通常不是排障重点 |

理解重点：遇到内存问题先分区定位，不要直接“加大内存”了事。

## 四、练习任务

1. 写一个主线程等待子线程完成的程序，观察执行顺序。
2. 用共享变量模拟并发写，比较“无同步”和“有同步”的输出差异。
3. 用自己的话写出“堆、栈、方法区”的职责和常见故障。

## 五、错答与修正（章级题解）

### 题目

`synchronized` 和 `volatile` 有什么区别？

### 错答示例（低分）

- “`volatile` 更轻量，所以可以替代 `synchronized`。”

### 修正答法（高分）

1. 先给结论：`volatile` 不能替代锁。
2. 再讲边界：`volatile` 只保证可见性与有序性，不保证复合操作原子性。
3. 再落地场景：状态开关可用 `volatile`，计数累加要用锁或原子类。

### 打分差异

| 维度 | 低分答法 | 高分答法 |
|---|---|---|
| 概念准确 | 以轻量替代功能边界 | 明确语义边界 |
| 场景匹配 | 一刀切替代 | 分场景选型 |
| 风险意识 | 不提竞态 | 提到丢更新风险 |

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
public class Main {
    // 这里故意不用 volatile 或锁，先演示“可能的竞态风险”
    private static int unsafeCounter = 0;

    public static void main(String[] args) throws InterruptedException {
        // 线程 A：执行 100000 次自增
        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 100000; i++) {
                // ++ 是读-改-写复合操作，不是原子操作
                unsafeCounter++;
            }
        });

        // 线程 B：同样执行 100000 次自增
        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 100000; i++) {
                unsafeCounter++;
            }
        });

        t1.start();
        t2.start();

        // join 让主线程等待两个工作线程结束，确保结果可观测
        t1.join();
        t2.join();

        // 理想值应为 200000，但实际在并发下可能小于该值
        System.out.println("unsafeCounter = " + unsafeCounter);
        System.out.println("理论值 = 200000（若小于该值，说明发生竞态）");
    }
}
```

**预期输出示例：**

```text
unsafeCounter = 183742
理论值 = 200000（若小于该值，说明发生竞态）
```
