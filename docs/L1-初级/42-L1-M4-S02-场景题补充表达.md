# L1-M4-S02 场景题补充表达

## 一句话结论

- 场景题补充表达 是 L1 阶段的关键能力点，面试回答建议覆盖“定义、原理、场景、边界”。

## 结构图

```mermaid
flowchart LR
    A[结论先行] --> B[原理展开]
    B --> C[项目场景]
    C --> D[边界追问]
```

## 核心知识点

1. 先给一句话结论，再展开关键点。
2. 回答要带项目场景和数据，不只背定义。
3. 提前准备追问分支，避免被问深后失焦。

## 高频面试题

### Q1：你如何在项目中落地“场景题补充表达”？

答题骨架：
1. 先说明业务目标和约束。
2. 再给可执行方案和关键指标。
3. 最后补充风险、边界与回退策略。

### Q2：场景题补充表达 的常见误区是什么？

答题骨架：
1. 说明常见错误做法。
2. 给出正确实践和适用条件。
3. 用一个真实场景收尾。


## 前置知识

- 能回答基础概念题。
- 知道至少 1 个自己参与的项目场景。
- 会描述问题-动作-结果。

## 术语解释（零基础友好）

- **场景题**：关注你如何在真实业务中解决问题。
- **闭环表达**：问题、分析、方案、结果、复盘完整串联。
- **量化指标**：可度量结果，如 RT、QPS、错误率。

## 详细学习步骤（从不会到会）

1. 先描述场景背景和约束，不急着给方案。
2. 再说你的分析过程和选择依据。
3. 给出落地动作和最终量化结果。
4. 最后补一条“如果重来会怎么优化”。

## 常见错误与纠偏

- 只讲做了什么，不讲为什么这样做。
- 结果没有数据支撑，可信度不足。

## 学习动作

- 先手敲一次示例代码，确保可以独立运行。
- 用自己的话复述“定义 -> 原理 -> 场景 -> 边界”。
- 把本节关键结论写成 3 句速记卡，第二天复盘。

## 练习任务（建议动手）

1. 准备 2 个场景：性能优化与稳定性治理。
2. 每个场景写 5 句版和 12 句版两个版本。

## 练习参考方向

- 5 句版用于快问快答，12 句版用于深挖追问。
- 每个场景至少准备 1 个可量化指标。

## 复习检查

- [ ] 能在 90 秒内说明本节核心结论
- [ ] 能独立运行并解释示例代码输出
- [ ] 能说出至少 1 个常见错误与修正方式
