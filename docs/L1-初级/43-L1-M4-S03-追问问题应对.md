# L1-M4-S03 追问问题应对

## 一句话结论

- 追问问题应对 是 L1 阶段的关键能力点，面试回答建议覆盖“定义、原理、场景、边界”。

## 结构图

```mermaid
flowchart LR
    A[结论先行] --> B[原理展开]
    B --> C[项目场景]
    C --> D[边界追问]
```

## 核心知识点

1. 先给一句话结论，再展开关键点。
2. 回答要带项目场景和数据，不只背定义。
3. 提前准备追问分支，避免被问深后失焦。

## 高频面试题

### Q1：你如何在项目中落地“追问问题应对”？

答题骨架：
1. 先说明业务目标和约束。
2. 再给可执行方案和关键指标。
3. 最后补充风险、边界与回退策略。

### Q2：追问问题应对 的常见误区是什么？

答题骨架：
1. 说明常见错误做法。
2. 给出正确实践和适用条件。
3. 用一个真实场景收尾。


## 前置知识

- 具备基础题和场景题表达能力。
- 知道常见追问方向（为什么/边界/替代方案）。
- 能在压力下保持结构化表达。

## 术语解释（零基础友好）

- **追问**：面试官基于你回答继续深挖细节的问题。
- **边界条件**：方案有效的前提与失效场景。
- **替代方案**：在不同约束下可选的其他实现。

## 详细学习步骤（从不会到会）

1. 先复述追问关键字，确认你理解问题。
2. 回答时先给结论，再补边界和取舍。
3. 遇到不会的点，坦诚说明并给可验证思路。
4. 最后回到主线，避免被追问带偏。

## 常见错误与纠偏

- 追问时急于给答案，忽略问题边界。
- 不会时硬编细节，容易被继续追问击穿。

## 学习动作

- 先手敲一次示例代码，确保可以独立运行。
- 用自己的话复述“定义 -> 原理 -> 场景 -> 边界”。
- 把本节关键结论写成 3 句速记卡，第二天复盘。

## 练习任务（建议动手）

1. 为同一道题准备 3 个追问分支及答案。
2. 模拟“不会某细节”时的稳妥回答模板。

## 练习参考方向

- 追问分支建议：原理、场景、替代方案。
- 不会时可说“我当前做法是…，后续会按…验证”。

## 复习检查

- [ ] 能在 90 秒内说明本节核心结论
- [ ] 能独立运行并解释示例代码输出
- [ ] 能说出至少 1 个常见错误与修正方式
