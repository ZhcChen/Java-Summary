# 01 技术方法论与架构决策

## 目标

- 把“技术好坏”转换成“业务价值 + 风险 + 成本”的统一决策框架。

## 二级菜单

1. 目标对齐：先定义业务结果，再谈技术方案。
2. 约束识别：时间、人力、预算、合规、稳定性。
3. 方案对比：至少两套方案，写清收益与代价。
4. 决策记录：保留 ADR（Architecture Decision Record）与回滚预案。
