## 提炼结论

### 1) JavaGuide：覆盖最全，适合作为主干

目录信号（docs 下 markdown 数量较高的模块）：
- `java`、`database`、`cs-basics`、`system-design`、`distributed-system`

结论：
- 适合作为“基础 + 面试 + 系统设计”的主线参考。

### 2) advanced-java：中高阶专题聚焦强

高权重目录：
- `high-concurrency`
- `distributed-system`
- `high-availability`
- `micro-services`

结论：
- 适合作为 L2/L3 的专题深挖素材。

### 3) source-code-hunter：源码层能力补齐

高权重目录：
- `Spring`、`Netty`、`Mybatis`、`Dubbo`、`JDK`

结论：
- 适合高级阶段的“原理追问”和“源码细节”准备。

### 4) developer-roadmap：导航组织方式值得借鉴

结论：
- 适合作为索引体验和路线表达参考，不直接替代 Java 内容。

## 结构化落地策略

基于以上调研，当前仓库采用：
- **一份总大纲**：`10-Java后端-知识大纲-v1.md`
- **三类索引**：学习顺序、面试频率、专题
- **三级内容**：L1/L2/L3 各 4 个核心模块
