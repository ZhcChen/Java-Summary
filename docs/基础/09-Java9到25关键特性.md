# 09 Java 9 到 25 关键特性

## 本章目标

- 建立 Java 9~25 的主线认知，而不是零散记忆。
- 理解“哪些特性该优先落地，哪些特性先观望”。
- 掌握版本升级时“语言层 + 库层 + 运行层”的评估方法。

## 前置知识

- 已掌握 Java 8 常用特性。
- 理解项目构建和运行环境的基本概念。

## 一、版本演进主线（基础视角）

> 以 2026 年视角看，企业常见 LTS 路线是：11 -> 17 -> 21 -> 25。

### 1) 9~11：平台能力升级起点

- 模块化（JPMS）
- `var` 局部变量类型推断（10）
- 标准 API 持续完善（HTTP Client 在 11 正式）

### 2) 12~17：语法表达力明显增强

- Switch 表达式（14）
- Text Blocks（15）
- Record（16）
- Pattern Matching for `instanceof`（16）
- 17 成为 LTS

### 3) 18~21：并发模型与生产可用性提升

- 虚拟线程（21 正式）
- 结构化并发/作用域值（在后续版本持续演进）
- 21 成为 LTS

### 4) 22~25：延续表达力与并发能力演进

- 模式匹配体系继续完善
- 并发工具链持续增强
- 25 成为最新 LTS 主线（适合新项目基线规划）

## 二、工程落地优先级建议

1. **优先落地已正式、团队可读的新语法**：record、switch 表达式、text blocks。
2. **并发特性先小范围试点**：如虚拟线程，先在非核心链路压测。
3. **预览/孵化特性谨慎使用**：避免核心链路深度绑定。

## Java 示例代码（需 JDK 17+，JDK 25 可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
public class Main {
    // record 减少数据载体样板代码（构造、equals/hashCode、toString 自动生成）
    record User(long id, String name, int level) {}

    static String levelName(int level) {
        // switch 表达式：分支更紧凑，避免漏写 break
        return switch (level) {
            case 1 -> "BEGINNER";
            case 2 -> "INTERMEDIATE";
            case 3 -> "ADVANCED";
            default -> "UNKNOWN";
        };
    }

    static String describe(Object obj) {
        // instanceof 模式匹配：减少显式强转样板
        if (obj instanceof User user) {
            return "User(" + user.id() + "," + user.name() + ")";
        }
        return "Other(" + obj + ")";
    }

    public static void main(String[] args) {
        User user = new User(1L, "alice", 2);

        // text block：多行文本更清晰
        String json = """
                {
                  "id": %d,
                  "name": "%s",
                  "level": "%s"
                }
                """.formatted(user.id(), user.name(), levelName(user.level()));

        System.out.println("user=" + user);
        System.out.println("desc=" + describe(user));
        System.out.println(json);
    }
}
```

**预期输出示例：**

```text
user=User[id=1, name=alice, level=2]
desc=User(1,alice)
{
  "id": 1,
  "name": "alice",
  "level": "INTERMEDIATE"
}
```

## 常见误区与修正

1. **误区：升级后马上全量改新语法**  
   修正：优先改收益高、风险低的模块，逐步推进。

2. **误区：预览特性直接上核心生产链路**  
   修正：先做隔离试点与回滚预案。

3. **误区：只关注语法，不关注依赖兼容性**  
   修正：升级是系统工程，需整体评估。

## 面试高频问答

### Q1：为什么企业偏好 LTS 版本？

- 一句话结论：LTS 维护周期更长，稳定性和生态兼容性更高。
- 关键点：
  1. 风险可控，升级节奏可规划。
  2. 依赖与框架支持更成熟。
  3. 便于统一团队开发基线。

### Q2：Java 新特性如何决定“能不能上生产”？

- 一句话结论：看成熟度、团队认知、业务风险和回滚成本。
- 关键点：
  1. 是否正式特性。
  2. 是否有替代方案。
  3. 是否完成压测与灰度验证。
