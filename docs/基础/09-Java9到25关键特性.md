# 09 Java 9 到 25 关键特性

## 本章目标

- 了解 Java 8 之后到 Java 25 的主线演进。
- 知道哪些特性能优先落地到工程代码。
- 明确预览特性与正式特性的使用边界。

## 前置知识

- 已掌握 Java 8 常用特性（Lambda、Stream、Optional、java.time）。
- 了解项目构建与运行环境的基础概念。

## 核心概念

1. Java 采用 Feature Release 节奏，LTS 版本更适合企业生产。
2. 版本升级要分“语言层、库层、运行层”三部分评估。
3. 新特性使用要考虑团队认知成本与维护成本。

## 原理展开

- **11/17/21/25** 是长期支持（LTS）主线。
- 9~17 重点提升语言表达力和平台能力；21 重点提升并发模型（如虚拟线程）。
- 22~25 延续并发、模式匹配与平台能力演进，含预览/孵化内容。

## 典型场景

- 老系统升级：8 -> 17（稳定过渡）或 8 -> 21/25（长期规划）。
- 新系统建设：优先以最新 LTS 为基线，减少后续大版本迁移成本。

## 常见误区

1. 一升级就全量启用所有新语法。
2. 预览特性直接用于核心生产链路。
3. 不做压测和灰度，直接全量上线。

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
public class Main {
    // record 在现代 Java 中可显著减少数据载体样板代码
    record User(long id, String name) {}

    static String levelName(int level) {
        // switch 表达式可提升分支代码可读性
        return switch (level) {
            case 1 -> "BEGINNER";
            case 2 -> "INTERMEDIATE";
            case 3 -> "ADVANCED";
            default -> "UNKNOWN";
        };
    }

    public static void main(String[] args) {
        User user = new User(1L, "alice");

        // text block 提升多行模板字符串可读性
        String profile = """
                {
                  "id": %d,
                  "name": "%s",
                  "level": "%s"
                }
                """.formatted(user.id(), user.name(), levelName(2));

        System.out.println(user);
        System.out.println(profile);
    }
}
```

**预期输出示例：**

```text
User[id=1, name=alice]
{
  "id": 1,
  "name": "alice",
  "level": "INTERMEDIATE"
}
```

## 练习与自测

1. 把旧 POJO 改写成 `record`。
2. 把多分支 if-else 改写为 switch 表达式。
3. 自测：能否说清“为什么企业更偏向 LTS 版本”。

## 本章小结

- 版本升级是工程治理问题，不只是语法更新。
- 先做稳定升级，再做新特性渐进落地。
