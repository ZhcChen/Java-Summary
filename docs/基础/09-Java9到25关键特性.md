# 09 Java 9 到 25 关键特性

## 本章目标

- 了解 Java 8 之后到 Java 25 的主线演进。
- 识别哪些特性可以直接提升工程可读性与可维护性。
- 避免“升级了 JDK，但开发习惯仍停留在旧版本”。

## 一、版本节奏与定位

- Java 从 9 开始采用 6 个月一个 Feature Release 的节奏。
- LTS 版本重点关注长期稳定：11、17、21、25。
- 基础学习建议：先吃透 8，再补 11/17/21，最后看 22~25 增量。

## 二、Java 9~11（过渡与稳定）

### 1) JPMS 模块化（Java 9）

适合大型项目做边界治理；小型项目不必强行引入。

### 2) `var`（Java 10）

- 优点：减少样板类型声明。
- 边界：只在局部变量使用，且变量名必须语义清晰。

### 3) 标准 HttpClient（Java 11）

统一的 HTTP 调用能力，API 更现代。

## 三、Java 12~17（语言表达力增强）

### 1) Switch 表达式（演进后可用）

分支表达更紧凑，减少漏写 `break` 风险。

### 2) Text Block

多行字符串可读性明显提升，适合 SQL/JSON 模板。

### 3) Record（Java 16 正式）

不可变数据载体更简洁，减少大量样板代码。

### 4) Sealed Class（Java 17）

限制继承范围，明确扩展边界。

## 四、Java 18~21（并发与集合能力增强）

### 1) Virtual Threads（Java 21）

提升高并发 I/O 场景下的并发模型表达力，降低线程管理复杂度。

### 2) Sequenced Collections（Java 21）

统一有序集合的首尾访问语义，API 更一致。

### 3) 模式匹配能力持续增强

`instanceof` 与 `switch` 的模式匹配能力更完整，减少模板判断代码。

## 五、Java 22~25（持续演进）

- 继续沿着 Loom、语言模式增强、外部内存/原生互操作等方向推进。
- 22~25 含大量预览/孵化特性，生产使用前需评估稳定性与团队认知成本。
- 在企业项目中，优先采用已正式定稿且团队可维护的特性。

## 六、升级建议

1. 先统一编译与运行时版本。
2. 再做语法升级（record、switch 表达式等）。
3. 最后做性能与运维层验证。

## 七、练习任务

1. 把一个旧 POJO 改写为 `record`。
2. 把多分支 if-else 改写为 switch 表达式。
3. 写一份“从 Java 8 升级到 21/25”的风险清单。

## Java 示例代码（含注释，可直接运行，JDK17+）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
public class Main {
    // Record: 不可变数据结构，减少模板代码
    record User(long id, String name) {}

    static String levelName(int level) {
        // Switch 表达式：写法更紧凑，默认分支必须覆盖
        return switch (level) {
            case 1 -> "BEGINNER";
            case 2 -> "INTERMEDIATE";
            case 3 -> "ADVANCED";
            default -> "UNKNOWN";
        };
    }

    public static void main(String[] args) {
        User user = new User(1L, "alice");

        // Text Block: 多行字符串可读性更好
        String profile = """
                {
                  "id": %d,
                  "name": "%s",
                  "level": "%s"
                }
                """.formatted(user.id(), user.name(), levelName(2));

        System.out.println(user);
        System.out.println(profile);
    }
}
```

**预期输出示例：**

```text
User[id=1, name=alice]
{
  "id": 1,
  "name": "alice",
  "level": "INTERMEDIATE"
}
```
