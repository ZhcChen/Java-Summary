# 02 集合与泛型

## 本章目标

- 能根据业务访问模式选择集合，而不是只背 API 名称。
- 理解泛型不变性与 PECS 原则，避免类型安全漏洞。
- 建立“先正确、再性能”的集合使用习惯。

## 前置知识

- 理解接口与实现类的关系。
- 会写基本循环和方法。
- 知道 `List`、`Set`、`Map` 的基础语义。

## 一、集合选型先看业务，不先看复杂度

### 1) 典型选型思路

1. 是否需要按下标访问？需要则优先 `ArrayList`。
2. 是否需要去重？需要则优先 `HashSet` / `LinkedHashSet`。
3. 是否需要键值检索？需要则选择 `HashMap` / `TreeMap`。
4. 是否存在并发写？存在则优先并发容器（如 `ConcurrentHashMap`）。

### 2) 为什么“看场景”比“背 O(1)/O(logN)”更重要

- 数据规模很小时，常数项可能比复杂度更关键。
- 对象创建、GC、缓存命中率都会影响真实性能。
- 读多写少、写多读少、是否有序，往往决定最终选型。

## 二、泛型的核心价值：把错误前移到编译期

### 1) 不变性（Invariance）

`List<Object>` 不是 `List<String>` 的父类型。  
原因：如果允许向 `List<String>` 写入 `Integer`，类型安全会被破坏。

### 2) PECS 原则

- `? extends T`：生产者，适合读，不适合写。
- `? super T`：消费者，适合写，读出来类型更宽。

### 3) 类型擦除的影响

- 泛型主要在编译期生效，运行时会做类型擦除。
- 不能直接 `new T()`，也不能安全地判断 `instanceof List<String>`。

## 三、Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
import java.util.ArrayList;
import java.util.HashMap;
import java.util.LinkedHashSet;
import java.util.List;
import java.util.Map;
import java.util.Set;

public class Main {
    // extends: 只读生产者，适合聚合计算
    static double sumNumbers(List<? extends Number> numbers) {
        double total = 0;
        for (Number n : numbers) {
            total += n.doubleValue();
        }
        return total;
    }

    // super: 只写消费者，适合批量写入
    static void addDefaultScores(List<? super Integer> target) {
        target.add(60);
        target.add(80);
        target.add(100);
    }

    // 通用复制：src 生产数据，dest 消费数据
    static <T> void copy(List<? super T> dest, List<? extends T> src) {
        for (T item : src) {
            dest.add(item);
        }
    }

    public static void main(String[] args) {
        // 1) List：保持插入顺序，支持按索引访问
        List<Integer> scoreList = new ArrayList<>();
        scoreList.add(95);
        scoreList.add(88);
        scoreList.add(95);

        // 2) Set：天然去重
        Set<Integer> uniqueScores = new LinkedHashSet<>(scoreList);

        // 3) Map：通过 key 快速检索
        Map<Long, String> userMap = new HashMap<>();
        userMap.put(1L, "alice");
        userMap.put(2L, "bob");

        // 4) 泛型通配符应用
        List<Number> reportNumbers = new ArrayList<>();
        addDefaultScores(reportNumbers);
        copy(reportNumbers, scoreList);

        System.out.println("scoreList=" + scoreList);
        System.out.println("uniqueScores=" + uniqueScores);
        System.out.println("user#1=" + userMap.get(1L));
        System.out.println("sum(scoreList)=" + sumNumbers(scoreList));
        System.out.println("reportNumbers=" + reportNumbers);
    }
}
```

**预期输出示例：**

```text
scoreList=[95, 88, 95]
uniqueScores=[88, 95]
user#1=alice
sum(scoreList)=278.0
reportNumbers=[60, 80, 100, 95, 88, 95]
```

## 四、常见误区与修正

1. **误区：所有列表都用 `LinkedList` 更灵活**  
   修正：大多数业务以读为主，`ArrayList` 更常见且性能稳定。

2. **误区：为了快，直接用原始类型 `List`**  
   修正：会丢失编译期检查，运行时更容易 `ClassCastException`。

3. **误区：并发环境里 `HashMap` 也能凑合用**  
   修正：并发写入请使用并发容器。

## 五、面试高频问答

### Q1：`ArrayList` 和 `LinkedList` 怎么选？

- 一句话结论：默认优先 `ArrayList`，只有明确需要链表特性时再选 `LinkedList`。
- 关键点：
  1. `ArrayList` 连续内存，随机访问快。
  2. `LinkedList` 节点分散，缓存命中率低。
  3. 真实业务里随机访问和遍历更常见。

### Q2：`? extends` 与 `? super` 的记忆方式？

- 一句话结论：PECS——Producer Extends，Consumer Super。
- 关键点：
  1. 读数据用 `extends`。
  2. 写数据用 `super`。
  3. 同时读写通常需要具体类型参数 `<T>`。

## 本章小结

- 集合选型先看业务行为，再看理论复杂度。
- 泛型是代码可维护性的基础设施，不是可选语法。
- PECS 是泛型实战最重要的判断规则。
