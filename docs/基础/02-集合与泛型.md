# 02 集合与泛型

## 本章目标

- 能按场景选择合适的集合结构。
- 理解泛型约束的边界，避免类型安全问题。
- 在性能与可读性之间做基本权衡。

## 前置知识

- 理解 `List`、`Set`、`Map` 的基本用途。
- 知道类和接口的关系。

## 核心概念

1. **集合选型看场景**：读写比例、是否去重、是否并发。
2. **泛型是类型契约**：把错误前移到编译期。
3. **通配符表达边界**：`? extends` 读多写少，`? super` 写多读少。

## 原理展开

- `ArrayList` 连续内存，随机访问快；`LinkedList` 在实际业务中不一定更快。
- `HashMap` 平均查询效率高，但并发写入场景应改用 `ConcurrentHashMap`。
- PECS 原则（Producer Extends, Consumer Super）是泛型实战关键。

## 典型场景

- 标签列表：`List<String>`。
- 用户去重：`Set<Long>`。
- 用户索引：`Map<Long, User>`。
- 并发计数映射：`ConcurrentHashMap<String, AtomicInteger>`。

## 常见误区

1. 只记复杂度，不结合数据规模和访问模式。
2. 使用 raw type（原始类型）绕开编译器检查。
3. 在并发写场景继续使用 `HashMap`。

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

public class Main {
    // extends: 生产者，安全读取
    static double sum(List<? extends Number> numbers) {
        double total = 0;
        for (Number n : numbers) {
            total += n.doubleValue();
        }
        return total;
    }

    // super: 消费者，安全写入 Integer
    static void addDefaults(List<? super Integer> target) {
        target.add(10);
        target.add(20);
    }

    public static void main(String[] args) {
        List<Integer> scores = new ArrayList<>();
        scores.add(95);
        scores.add(88);

        Map<Long, String> userMap = new HashMap<>();
        userMap.put(1L, "alice");

        List<Number> numbers = new ArrayList<>();
        addDefaults(numbers);

        System.out.println("sum(scores)=" + sum(scores));
        System.out.println("numbers=" + numbers);
        System.out.println("user1=" + userMap.get(1L));
    }
}
```

**预期输出示例：**

```text
sum(scores)=183.0
numbers=[10, 20]
user1=alice
```

## 练习与自测

1. 给“订单列表、去重集合、索引映射”写选型说明。
2. 实现 `copy(List<? super T> dest, List<? extends T> src)` 方法。
3. 自测：能否解释 `? extends` 为什么不能安全写入。

## 本章小结

- 集合选型要以场景为先，不要只背 API。
- 泛型是可维护性的基础设施，不是可有可无的语法。
