# 02 集合与泛型

## 本章目标

- 能根据场景选择合适集合。
- 理解泛型的边界约束，避免类型安全问题。
- 避免常见集合误用导致的性能或并发问题。

## 一、集合选型核心思路

先回答三个问题：

1. 数据是否需要去重？
2. 是否需要按键快速查找？
3. 是否存在并发读写？

据此选择：

| 需求 | 常用结构 | 说明 |
|---|---|---|
| 顺序访问、按下标读多 | `ArrayList` | 连续内存，随机访问快 |
| 去重 | `HashSet` | 底层哈希实现 |
| 键值映射 | `HashMap` | 平均 O(1) 查找 |
| 并发 Map | `ConcurrentHashMap` | 并发场景更安全 |
| 排序键值 | `TreeMap` | 支持范围查询 |

## 二、泛型不是语法糖，而是“类型契约”

### 1) 为什么有泛型

- 编译期发现类型错误，避免运行时 `ClassCastException`。
- API 表达更明确，调用者更容易理解约束。

### 2) 通配符规则

- `? extends T`：读多写少（生产者）。
- `? super T`：写多读少（消费者）。

记忆方式：PECS（Producer Extends, Consumer Super）。

## 三、常见错误

1. 在并发写场景使用 `HashMap`。
2. 滥用原始类型（raw type），绕过泛型检查。
3. 只看理论复杂度，不看实际数据规模与访问模式。

## 四、练习任务

1. 为“用户标签、订单索引、实时计数”三个场景做集合选型。
2. 写一个方法，接收 `List<? extends Number>` 并求和。
3. 把一个 raw type 集合改造成泛型安全版本。

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

public class Main {
    // 读取 Number 的子类型集合，适合使用 extends
    static double sum(List<? extends Number> numbers) {
        double total = 0;
        for (Number n : numbers) {
            total += n.doubleValue();
        }
        return total;
    }

    // 向 Number 的父类型集合写入 Integer，适合使用 super
    static void addDefaults(List<? super Integer> target) {
        target.add(10);
        target.add(20);
    }

    public static void main(String[] args) {
        // 场景 1：顺序列表
        List<Integer> scores = new ArrayList<>();
        scores.add(95);
        scores.add(88);

        // 场景 2：键值映射
        Map<Long, String> userMap = new HashMap<>();
        userMap.put(1L, "alice");

        List<Number> numbers = new ArrayList<>();
        addDefaults(numbers);

        System.out.println("sum(scores)=" + sum(scores));
        System.out.println("numbers=" + numbers);
        System.out.println("user1=" + userMap.get(1L));
    }
}
```

**预期输出示例：**

```text
sum(scores)=183.0
numbers=[10, 20]
user1=alice
```
