# 03 字符串、数值与常用基础库

## 本章目标

- 避免字符串比较、金额计算、日期处理中的高频错误。
- 理解 `String`、`BigDecimal`、`Optional`、`java.time` 的正确边界。
- 形成“语义正确优先”的基础编码习惯。

## 前置知识

- 理解基本类型与引用类型区别。
- 会写方法和条件判断。
- 知道 `null` 的基本语义。

## 一、字符串处理：先保证语义，再考虑简写

### 1) `String` 的关键事实

- `String` 不可变：每次拼接都会生成新对象。
- 比较内容要用 `equals`，不要用 `==`。
- 高频拼接优先 `StringBuilder`，减少中间对象。

### 2) 字符编码为什么重要

- 业务常见输入输出（日志、文件、接口）都依赖编码。
- 未显式指定编码会受环境影响，导致“本地正常、线上乱码”。

## 二、金额处理：业务正确性 > 写法省事

### 1) 为什么不用 `double` 处理金额

`double` 是二进制浮点，很多十进制小数无法精确表示。  
金额场景需要“可控舍入 + 可重复计算”，应使用 `BigDecimal`。

### 2) `BigDecimal` 的两个高频规则

1. 构造优先用字符串：`new BigDecimal("0.1")`。
2. 除法显式指定舍入规则：`divide(..., RoundingMode.HALF_UP)`。

## 三、可空值与时间处理

### 1) Optional 的定位

- 适合用于“返回值语义表达”，表示“可能没有值”。
- 不建议滥用为实体字段或序列化字段。

### 2) java.time 的优势

- API 语义更清晰，线程安全。
- 比旧 `Date/Calendar` 更易维护和测试。

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
import java.math.BigDecimal;
import java.math.RoundingMode;
import java.time.LocalDateTime;
import java.time.ZoneId;
import java.time.ZonedDateTime;
import java.time.format.DateTimeFormatter;
import java.util.Optional;

public class Main {
    static BigDecimal calculatePayable(BigDecimal originAmount, BigDecimal discountRate, BigDecimal taxRate) {
        // 折后价 = 原价 * 折扣
        BigDecimal discounted = originAmount.multiply(discountRate);

        // 税额 = 折后价 * 税率
        BigDecimal tax = discounted.multiply(taxRate);

        // 显式指定小数位和舍入规则，避免不同环境默认行为不一致
        return discounted.add(tax).setScale(2, RoundingMode.HALF_UP);
    }

    public static void main(String[] args) {
        // 1) String 比较必须用 equals
        String fromDb = new String("VIP");
        String fromRule = "VIP";
        boolean isVip = fromRule.equals(fromDb);

        // 2) 金额计算必须使用 BigDecimal（字符串构造）
        BigDecimal origin = new BigDecimal("199.90");
        BigDecimal discountRate = new BigDecimal("0.85");
        BigDecimal taxRate = new BigDecimal("0.06");
        BigDecimal payable = calculatePayable(origin, discountRate, taxRate);

        // 3) Optional 用于返回值缺省
        String nickName = Optional.<String>ofNullable(null).orElse("guest");

        // 4) 使用 java.time 处理时区时间
        ZonedDateTime shanghaiNow = ZonedDateTime.now(ZoneId.of("Asia/Shanghai"));
        LocalDateTime expireAt = shanghaiNow.plusMinutes(30).toLocalDateTime();

        System.out.println("isVip=" + isVip);
        System.out.println("payable=" + payable);
        System.out.println("nickName=" + nickName);
        System.out.println("expireAt=" + expireAt.format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")));
    }
}
```

**预期输出示例：**

```text
isVip=true
payable=180.11
nickName=guest
expireAt=2026-... ...
```

## 常见误区与修正

1. **误区：`==` 比字符串更快，所以可用于内容比较**  
   修正：`==` 比较引用地址，不比较内容。

2. **误区：`new BigDecimal(0.1)` 也没问题**  
   修正：构造阶段就可能引入误差。

3. **误区：Optional 能减少空指针，就 everywhere 使用**  
   修正：Optional 是语义工具，不是全局替代品。

## 面试高频问答

### Q1：为什么金额不用 `double`？

- 一句话结论：`double` 在十进制金额场景会有不可控误差，`BigDecimal` 更可靠。
- 关键点：
  1. 二进制浮点无法精确表示多数十进制小数。
  2. 金额计算需要精确舍入。
  3. 财务场景要求可重复、可审计。

### Q2：Optional 的正确使用边界？

- 一句话结论：用于返回值表达“可能为空”，不宜滥用于字段和参数。
- 关键点：
  1. 表达语义清晰。
  2. 链式调用可减少判空噪音。
  3. 滥用会导致可读性下降。
