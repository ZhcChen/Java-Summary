# 03 字符串、数值与常用基础库

## 本章目标

- 避免字符串处理、金额计算和空值处理中的高频错误。
- 理解 `BigDecimal`、`Optional`、`java.time` 的正确使用方式。
- 建立“结果正确优先于写法简短”的基础习惯。

## 前置知识

- 理解基本类型与引用类型区别。
- 知道 `String`、`double`、`null` 的基础语义。

## 核心概念

1. `String` 不可变，频繁拼接优先 `StringBuilder`。
2. 金额计算优先 `BigDecimal`，避免浮点误差。
3. `Optional` 用于表达“可能为空”的语义。
4. 日期时间优先 `java.time`。

## 原理展开

- `double` 采用二进制浮点表示，十进制金额容易出现误差。
- `BigDecimal` 构造推荐字符串形式，避免误差在构造阶段引入。
- `Optional` 适合用于返回值语义，不建议滥用为实体字段。

## 典型场景

- 订单折扣计算、税费计算。
- 用户昵称缺失时使用默认值。
- 计算订单超时与时间窗口。

## 常见误区

1. 用 `==` 比较字符串内容。
2. 用 `new BigDecimal(0.1)` 处理金额。
3. 使用旧日期 API 导致线程安全和时区问题。

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
import java.math.BigDecimal;
import java.math.RoundingMode;
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;
import java.util.Optional;

public class Main {
    public static void main(String[] args) {
        // 金额计算：字符串构造避免二进制浮点误差
        BigDecimal price = new BigDecimal("199.90");
        BigDecimal discountRate = new BigDecimal("0.85");
        BigDecimal finalPrice = price.multiply(discountRate).setScale(2, RoundingMode.HALF_UP);

        // Optional 表达缺省值语义
        String nickname = Optional.<String>ofNullable(null).orElse("guest");

        // java.time 更清晰且线程安全
        LocalDate nextWeek = LocalDate.now().plusDays(7);
        String formattedDate = nextWeek.format(DateTimeFormatter.ISO_DATE);

        System.out.println("finalPrice=" + finalPrice);
        System.out.println("nickname=" + nickname);
        System.out.println("nextWeek=" + formattedDate);
    }
}
```

**预期输出示例：**

```text
finalPrice=169.92
nickname=guest
nextWeek=2026-03-12
```
