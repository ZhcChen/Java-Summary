# 03 字符串、数值与常用基础库

## 本章目标

- 理解字符串、金额计算、判空处理中的常见坑。
- 会使用 Java 基础库提升代码可读性和稳定性。
- 避免“能跑但不可靠”的基础实现。

## 一、字符串处理

### 1) `String` 是不可变对象

- 拼接频繁场景优先 `StringBuilder`。
- 不可变性带来线程安全和缓存复用优势。

### 2) 常见坑

- 用 `==` 比较字符串内容。
- 在循环中频繁 `+` 拼接，导致大量中间对象。

## 二、金额与数值处理

### 1) 为什么不用 `double` 算金额

`double` 是二进制浮点，十进制金额可能出现精度误差。

### 2) `BigDecimal` 使用要点

- 构造时优先字符串：`new BigDecimal("0.1")`。
- 除法要显式指定舍入规则。
- 比较数值大小使用 `compareTo`，不是 `equals`。

## 三、常用基础库

- `Objects`：空值校验、空安全比较。
- `Optional`：明确“值可能不存在”的语义。
- `java.time`：替代老旧日期 API，提升可读性和线程安全。

## 四、练习任务

1. 写一个金额折扣计算器，要求避免精度误差。
2. 用 `Optional` 重写一段多层判空逻辑。
3. 用 `java.time` 输出“当前时间 + 7 天”的格式化字符串。

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
import java.math.BigDecimal;
import java.math.RoundingMode;
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;
import java.util.Optional;

public class Main {
    public static void main(String[] args) {
        // 1) 金额计算：用字符串构造，避免二进制浮点误差
        BigDecimal price = new BigDecimal("199.90");
        BigDecimal discountRate = new BigDecimal("0.85");
        BigDecimal finalPrice = price.multiply(discountRate).setScale(2, RoundingMode.HALF_UP);

        // 2) Optional 表达“可能为空”的语义
        String nickname = Optional.<String>ofNullable(null).orElse("guest");

        // 3) java.time 处理日期
        LocalDate nextWeek = LocalDate.now().plusDays(7);
        String formattedDate = nextWeek.format(DateTimeFormatter.ISO_DATE);

        System.out.println("finalPrice=" + finalPrice);
        System.out.println("nickname=" + nickname);
        System.out.println("nextWeek=" + formattedDate);
    }
}
```

**预期输出示例：**

```text
finalPrice=169.92
nickname=guest
nextWeek=2026-03-12
```
