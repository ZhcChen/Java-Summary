# 11 基础综合练习与自测

## 本章目标

- 把前 10 章知识串成可执行能力链路。
- 用统一业务场景验证“概念 + 代码 + 表达”。
- 明确进入进阶前的能力缺口。

## 前置知识

- 已完成基础层 01~10 章节。
- 能独立运行并解释每章示例代码。

## 一、为什么要做综合练习

基础阶段最常见问题不是“某个知识点不会”，而是：

- 单章能看懂，跨章不会串联。
- 代码能跑通，但讲不清设计取舍。
- 知道理论，遇到故障缺少定位路径。

综合练习的目标就是把这些断点打通。

## 二、建议综合题（按重要性）

1. **用户查询链路**：集合缓存 + Optional 缺省 + 异常语义。
2. **并发统计任务**：线程池 + 原子计数 + 结果聚合。
3. **升级检查演练**：清单管理 + 验证阈值 + 发布决策。

## 三、综合能力评分建议（100 分）

- 概念准确（25 分）
- 原理解释（25 分）
- 代码质量（25 分）
- 风险与验证（25 分）

80 分以上可进入进阶层；低于 80 分建议回补薄弱章节。

## Java 示例代码（跨章节综合，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
import java.util.LinkedHashMap;
import java.util.Map;
import java.util.Optional;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;
import java.util.concurrent.atomic.AtomicInteger;

class User {
    private final long id;
    private final String name;

    User(long id, String name) {
        this.id = id;
        this.name = name;
    }

    long getId() {
        return id;
    }

    String getName() {
        return name;
    }
}

class UserRepository {
    private final Map<Long, User> data = new LinkedHashMap<>();

    UserRepository() {
        data.put(1L, new User(1L, "alice"));
        data.put(2L, new User(2L, "bob"));
    }

    Optional<User> findById(long id) {
        // Optional 显式表达“可能查询不到”
        return Optional.ofNullable(data.get(id));
    }
}

public class Main {
    public static void main(String[] args) throws InterruptedException {
        UserRepository repository = new UserRepository();

        // 1) 查询链路：成功路径 + 缺省路径
        String user1 = repository.findById(1L).map(User::getName).orElse("guest");
        String user9 = repository.findById(9L).map(User::getName).orElse("guest");
        System.out.println("user#1=" + user1);
        System.out.println("user#9=" + user9);

        // 2) 并发统计：线程池 + 原子计数
        ExecutorService pool = Executors.newFixedThreadPool(4);
        AtomicInteger successCounter = new AtomicInteger(0);

        for (int i = 0; i < 1000; i++) {
            long queryId = (i % 3 == 0) ? 1L : 9L;
            pool.submit(() -> {
                boolean hit = repository.findById(queryId).isPresent();
                if (hit) {
                    successCounter.incrementAndGet();
                }
            });
        }

        pool.shutdown();
        pool.awaitTermination(5, TimeUnit.SECONDS);

        // 3) 自评分卡：帮助学习闭环
        Map<String, Integer> scoreCard = new LinkedHashMap<>();
        scoreCard.put("概念准确", 24);
        scoreCard.put("原理解释", 22);
        scoreCard.put("代码质量", 23);
        scoreCard.put("风险与验证", 21);

        int total = scoreCard.values().stream().mapToInt(Integer::intValue).sum();
        scoreCard.forEach((k, v) -> System.out.println(k + "=" + v));

        System.out.println("hitCount=" + successCounter.get());
        System.out.println("totalScore=" + total);
        System.out.println(total >= 80 ? "result=PASS" : "result=REVIEW");
    }
}
```

**预期输出示例：**

```text
user#1=alice
user#9=guest
概念准确=24
原理解释=22
代码质量=23
风险与验证=21
hitCount=334
totalScore=90
result=PASS
```

> 说明：`hitCount` 可能因任务调度略有波动，但通常接近 334。

## 常见误区与修正

1. **误区：只做成功路径，不测失败分支**  
   修正：每个综合题至少补 1 个失败分支。

2. **误区：代码跑通就结束**  
   修正：还要解释取舍、边界和风险。

3. **误区：不复盘评分项**  
   修正：评分卡是定位短板的工具，不是形式。

## 面试高频问答

### Q1：如何证明你的基础能力是“成体系”的？

- 一句话结论：能把语法、集合、异常、并发、JVM 串成同一业务链路并解释取舍。
- 关键点：
  1. 给出可运行代码。
  2. 能说清设计边界。
  3. 能提出验证指标。

### Q2：基础阶段如何复习最有效？

- 一句话结论：按“章节回放 + 综合题演练 + 口述表达”三段法复习。
- 关键点：
  1. 章节复盘抓核心模型。
  2. 综合题验证跨章能力。
  3. 口述训练增强面试与协作表达。
