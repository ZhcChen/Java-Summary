# 11 基础综合练习与自测

## 使用方式

1. 先按章节完成练习。
2. 再做综合题，不查资料先作答。
3. 最后对照评分标准补齐薄弱点。

## 综合题 1：用户查询链路

场景：给定用户 ID，先查缓存，未命中再查数据库并回填缓存。

考察点：

- 基础分层设计
- 集合与 Map 使用
- 异常处理与空值处理

## 综合题 2：并发计数与线程池

场景：统计并发请求总数，要求结果准确，不能丢计数。

考察点：

- 线程安全
- 线程池基本配置
- 结果验证

## 综合题 3：版本升级前检查

场景：项目从 Java 8 升级到 17，设计上线前检查清单。

考察点：

- 升级风险识别
- 回归与压测验证
- 回滚策略

## 自评分标准（100 分）

| 维度 | 分值 |
|---|---:|
| 概念准确 | 25 |
| 原理完整 | 25 |
| 代码质量 | 25 |
| 风险与验证 | 25 |

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
import java.util.LinkedHashMap;
import java.util.Map;

public class Main {
    public static void main(String[] args) {
        // 模拟综合题自评分
        Map<String, Integer> score = new LinkedHashMap<>();
        score.put("概念准确", 21);
        score.put("原理完整", 20);
        score.put("代码质量", 22);
        score.put("风险与验证", 18);

        int total = 0;
        for (int value : score.values()) {
            total += value;
        }

        System.out.println("total=" + total);
        System.out.println(total >= 80 ? "result=PASS" : "result=REVIEW");
    }
}
```

**预期输出示例：**

```text
total=81
result=PASS
```
