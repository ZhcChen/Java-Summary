# 基础学习路线（书籍版）

## 这份基础层怎么用

这部分不是“速查清单”，而是一条完整的学习主线。建议按章节顺序读完，做到以下三件事：

1. 能解释概念：不只背定义，能说清“为什么这么设计”。
2. 能运行代码：每章示例都手敲并运行，再做 1~2 处改动观察结果。
3. 能复盘表达：每章最后用 2~3 分钟复述“概念 -> 原理 -> 场景 -> 误区”。

## 学习目标（完成基础层后）

- 能独立写出结构清晰的 Java 基础代码。
- 能定位常见基础问题（空指针、并发丢数据、编码乱码、异常边界混乱）。
- 能看懂 Java 8 业务代码，并理解 Java 9~25 的关键增量特性。

## 三阶段主线

| 阶段 | 核心主题 | 对应章节 | 产出标准 |
|---|---|---|---|
| A 语言与建模 | 语法、面向对象、集合、基础库 | 01~03 | 能写可读、可维护的基础业务代码 |
| B 稳定性基础 | 异常、IO/NIO、并发、JVM | 04~07 | 能解释常见故障原因并给出修复方向 |
| C 版本与工程 | Java 8、9~25、升级策略、综合训练 | 08~11 | 能做版本规划和综合能力自测 |

## 每章固定阅读方法（建议保持一致）

1. 先看“本章目标”和“前置知识”，确认当前章节的边界。
2. 读“原理展开”，把术语转换成自己的话。
3. 运行“Java 示例代码”，关注代码注释解释的“原因”。
4. 完成“练习与自测”，把知识点转成可复用能力。

## 建议学习节奏（8 周示例）

- 第 1~2 周：01~03（语法、集合、基础库）
- 第 3~4 周：04~05（异常、IO）
- 第 5~6 周：06~07（并发、JVM）
- 第 7 周：08~10（版本特性与升级）
- 第 8 周：11（综合练习 + 复盘）

## Java 示例代码（学习进度追踪器）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
import java.util.LinkedHashMap;
import java.util.Map;

public class Main {
    public static void main(String[] args) {
        // 使用有序 Map 记录章节状态，保证输出顺序稳定，便于每日复盘
        Map<String, Boolean> chapterStatus = new LinkedHashMap<>();
        chapterStatus.put("01-Java语法与面向对象", true);
        chapterStatus.put("02-集合与泛型", true);
        chapterStatus.put("03-字符串数值与常用基础库", false);
        chapterStatus.put("04-异常机制与调试思维", false);

        long finished = chapterStatus.values().stream().filter(Boolean::booleanValue).count();
        long total = chapterStatus.size();

        System.out.println("基础层进度=" + finished + "/" + total);
        chapterStatus.forEach((chapter, done) ->
                System.out.println(chapter + " -> " + (done ? "DONE" : "TODO"))
        );

        // 小步推进：每次只追求“下一章完成”，而不是一次吃透全部章节
        if (finished < total) {
            System.out.println("nextAction=继续下一章并运行示例代码");
        } else {
            System.out.println("nextAction=进入进阶层学习");
        }
    }
}
```

**预期输出示例：**

```text
基础层进度=2/4
01-Java语法与面向对象 -> DONE
02-集合与泛型 -> DONE
03-字符串数值与常用基础库 -> TODO
04-异常机制与调试思维 -> TODO
nextAction=继续下一章并运行示例代码
```

## 常见误区

1. 只读不跑代码，导致“看懂了但不会写”。
2. 章节之间割裂学习，无法串联成完整链路。
3. 追求快，把基础问题留到进阶阶段才暴露。
