# 04 异常机制与调试思维

## 本章目标

- 理解异常的分类和边界。
- 会写可读、可追踪、可恢复的异常处理代码。
- 形成“先定位根因再修复”的调试习惯。

## 一、异常分类

### 1) Checked Exception

- 编译期强制处理，通常用于可恢复外部错误（如 IO）。

### 2) Runtime Exception

- 运行时异常，通常代表编程错误或业务规则不满足。

### 3) Error

- 严重系统错误（如内存不足），一般不在业务代码中捕获处理。

## 二、异常处理原则

1. 不吞异常：至少记录关键信息。
2. 不乱抛：只在边界层做统一转换。
3. 保留上下文：错误信息要带业务标识。
4. 先恢复再失败：能降级就降级，不能降级再失败返回。

## 三、调试思维（四步）

1. 复现：明确输入、环境、步骤。
2. 定位：看堆栈第一现场，不先猜。
3. 验证：修复后用同样输入回归。
4. 复盘：记录原因和防再发措施。

## 四、练习任务

1. 设计一个业务异常类，包含错误码和错误消息。
2. 用 `try-with-resources` 改造文件读取逻辑。
3. 写一份“异常定位清单”并应用在一个 bug 上。

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
class BizException extends RuntimeException {
    private final String code;

    BizException(String code, String message) {
        super(message);
        this.code = code;
    }

    String getCode() {
        return code;
    }
}

public class Main {
    static int parsePositiveInt(String raw) {
        try {
            int value = Integer.parseInt(raw);
            if (value <= 0) {
                // 业务规则不满足时，抛业务异常而不是返回魔法值
                throw new BizException("BIZ_001", "value must be positive");
            }
            return value;
        } catch (NumberFormatException e) {
            // 把底层异常转换为更可读的业务语义，并保留原信息
            throw new BizException("BIZ_002", "invalid number: " + raw);
        }
    }

    public static void main(String[] args) {
        String[] inputs = {"12", "-2", "abc"};
        for (String input : inputs) {
            try {
                int result = parsePositiveInt(input);
                System.out.println("ok=" + result);
            } catch (BizException e) {
                System.out.println("error code=" + e.getCode() + ", msg=" + e.getMessage());
            }
        }
    }
}
```

**预期输出示例：**

```text
ok=12
error code=BIZ_001, msg=value must be positive
error code=BIZ_002, msg=invalid number: abc
```
