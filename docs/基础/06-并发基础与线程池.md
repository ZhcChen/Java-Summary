# 06 并发基础与线程池

## 本章目标

- 理解并发问题本质：共享可变状态。
- 区分 `synchronized`、`volatile`、原子类的适用边界。
- 掌握线程池基础配置思路。

## 一、并发三要素

1. **可见性**：线程 A 修改的数据，线程 B 是否能立刻看到。
2. **原子性**：一组操作是否不可分割。
3. **有序性**：执行顺序是否符合预期。

## 二、常用并发手段

- `synchronized`：互斥 + 可见性。
- `volatile`：可见性 + 有序性（不保证复合操作原子性）。
- `AtomicInteger`：针对特定场景的原子操作。

## 三、线程池配置核心

- `corePoolSize`：常驻线程数。
- `maximumPoolSize`：最大线程数。
- `workQueue`：任务排队策略。
- `RejectedExecutionHandler`：过载保护策略。

思路：先确定目标吞吐和最大可接受延迟，再反推参数。

## 四、常见错误

1. 以为 `volatile` 可以解决所有并发问题。
2. 线程池队列无上限，最终把延迟问题变成内存问题。
3. 不设置拒绝策略，过载时行为不可控。

## 五、练习任务

1. 写一个计数器，分别用“无同步”和“有同步”对比结果。
2. 建一个固定线程池，执行 1000 个任务并统计耗时。
3. 模拟队列堆积，观察拒绝策略触发行为。

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;

public class Main {
    private static int unsafeCounter = 0;
    private static int safeCounter = 0;

    // 用 synchronized 保护复合操作，避免并发丢更新
    private static synchronized void increaseSafeCounter() {
        safeCounter++;
    }

    public static void main(String[] args) throws InterruptedException {
        ExecutorService pool = Executors.newFixedThreadPool(8);

        for (int i = 0; i < 10000; i++) {
            pool.submit(() -> {
                // 无保护自增：在并发场景可能丢数据
                unsafeCounter++;
                // 有保护自增：结果稳定
                increaseSafeCounter();
            });
        }

        pool.shutdown();
        pool.awaitTermination(5, TimeUnit.SECONDS);

        System.out.println("unsafeCounter=" + unsafeCounter);
        System.out.println("safeCounter=" + safeCounter);
    }
}
```

**预期输出示例：**

```text
unsafeCounter=97xx
safeCounter=10000
```
