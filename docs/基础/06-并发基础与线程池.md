# 06 并发基础与线程池

## 本章目标

- 理解并发问题本质与线程安全三要素。
- 区分 `synchronized`、`volatile`、原子类的适用边界。
- 掌握线程池参数的基础配置方法。

## 前置知识

- 会写循环和方法。
- 理解“多个任务同时执行”的基本概念。

## 核心概念

1. 可见性：线程间数据能否及时看到。
2. 原子性：复合操作是否不可分割。
3. 有序性：执行顺序是否符合预期。

## 原理展开

- `volatile` 不能保证复合操作原子性。
- `synchronized` 提供互斥 + 可见性，适合保护临界区。
- 线程池通过“线程复用 + 队列 + 拒绝策略”控制资源。

## 典型场景

- 并发计数统计。
- 异步任务批量执行。
- 接口请求过载保护。

## 常见误区

1. 认为 `volatile` 可以替代锁。
2. 线程池队列无上限，导致延迟和内存问题。
3. 不设置拒绝策略，过载行为不可预测。

## Java 示例代码（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;

public class Main {
    private static int unsafeCounter = 0;
    private static int safeCounter = 0;

    // 用 synchronized 保护复合操作
    private static synchronized void increaseSafeCounter() {
        safeCounter++;
    }

    public static void main(String[] args) throws InterruptedException {
        ExecutorService pool = Executors.newFixedThreadPool(8);

        for (int i = 0; i < 10000; i++) {
            pool.submit(() -> {
                unsafeCounter++;      // 并发下可能丢更新
                increaseSafeCounter(); // 结果稳定
            });
        }

        pool.shutdown();
        pool.awaitTermination(5, TimeUnit.SECONDS);

        System.out.println("unsafeCounter=" + unsafeCounter);
        System.out.println("safeCounter=" + safeCounter);
    }
}
```

**预期输出示例：**

```text
unsafeCounter=97xx
safeCounter=10000
```

## 练习与自测

1. 用 `AtomicInteger` 改写示例计数器。
2. 设计一个有界队列线程池并验证拒绝策略。
3. 自测：能否讲清“为什么 safeCounter 一定是 10000”。

## 本章小结

- 并发编程先保正确，再谈性能。
- 线程池是资源治理工具，不只是“开线程”。
