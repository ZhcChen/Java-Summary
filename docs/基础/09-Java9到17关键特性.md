# 09 Java 9 到 17 关键特性

## 本章目标

- 了解 Java 8 之后最值得优先掌握的语言与平台特性。
- 能识别哪些特性能立即提升工程可读性与可维护性。
- 避免“版本升级了，但写法仍停留在老习惯”。

## 一、Java 9~11（过渡与稳定）

### 1) 模块化（JPMS）

适用于大型项目边界治理，不建议在小项目中强行引入。

### 2) `var`（Java 10）

- 优点：减少样板类型声明。
- 边界：只在局部变量使用，且变量名要足够语义化。

### 3) 标准 HttpClient（Java 11）

可替代部分旧 HTTP 调用方案，接口更现代。

## 二、Java 14~17（语言表达力增强）

### 1) Record

适合不可变数据载体，自动生成构造器、访问器、`equals/hashCode/toString`。

### 2) Switch 表达式

比传统 `switch` 更紧凑，减少漏写 `break` 风险。

### 3) Text Block

多行字符串更可读，适合 SQL、JSON 模板。

### 4) Sealed Class

限制继承层级，明确可扩展边界。

## 三、升级建议

1. 先统一编译与运行时版本。
2. 再做语法升级（record、switch 表达式等）。
3. 最后做性能与运维层验证。

## 四、练习任务

1. 把一个 POJO 改写为 `record`。
2. 把 if-else 分支改写为 switch 表达式。
3. 用 text block 重写多行 SQL 字符串。

## Java 示例代码（含注释，可直接运行，JDK17）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
public class Main {
    // Record: 表达不可变数据结构，减少样板代码
    record User(long id, String name) {}

    static String levelName(int level) {
        // Switch 表达式：分支更紧凑，默认分支必须覆盖
        return switch (level) {
            case 1 -> "BEGINNER";
            case 2 -> "INTERMEDIATE";
            case 3 -> "ADVANCED";
            default -> "UNKNOWN";
        };
    }

    public static void main(String[] args) {
        User user = new User(1L, "alice");

        // Text Block: 多行文本更易读
        String profile = """
                {
                  "id": %d,
                  "name": "%s",
                  "level": "%s"
                }
                """.formatted(user.id(), user.name(), levelName(2));

        System.out.println(user);
        System.out.println(profile);
    }
}
```

**预期输出示例：**

```text
User[id=1, name=alice]
{
  "id": 1,
  "name": "alice",
  "level": "INTERMEDIATE"
}
```
