# L2-M4-S04 线上故障定位闭环

## 一句话结论

- 线上故障定位闭环 是 L2 阶段的关键能力点，面试回答建议覆盖“定义、原理、场景、边界”。

## 结构图

```mermaid

    A[日志] --> B[指标]

```

## 核心知识点

1. 可观测性目标是“快定位、快止血、快复盘”。
2. Logs/Metrics/Trace 要使用统一标识（traceId）串联。
3. 告警要防噪音，优先业务影响指标。

## 前置知识

- 会看基础日志。
- 知道请求链路经过多个服务。

## 术语解释（零基础友好）

- **Trace**：一次请求跨服务调用链。
- **Metrics**：可聚合统计指标。

## 详细学习步骤（从不会到会）

1. 统一 traceId 贯穿日志。
2. 先看指标发现异常，再追踪链路。
3. 最后通过日志定位根因并复盘。

## 常见错误与纠偏

- 只有日志没有指标。
- 告警过多无优先级。

## 练习任务（建议动手）

1. 给一个接口设计最小监控指标集。
2. 模拟慢请求定位流程并记录步骤。

## 练习参考方向

- 可观测性目标是“快定位+可复盘”。

## Java 示例代码（含注释，可直接运行）


**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

**预期输出（示例）：**
```text
traceId=trace-001 service=order msg=start
traceId=trace-001 service=pay msg=done
```

```java
public class Main {
    public static void main(String[] args) {
        // traceId 串联日志、指标与链路追踪
        String traceId = "trace-001";
        System.out.println("traceId=" + traceId + " service=order msg=start");
        System.out.println("traceId=" + traceId + " service=pay msg=done");
    }
}
```
