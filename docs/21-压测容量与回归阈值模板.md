# 压测、容量估算与回归阈值模板

> 目的：给学习者一套可复用的工程模板，避免“只讲理论、不落地”。

## 一、压测前输入模板

| 项目 | 示例值 | 说明 |
|---|---:|---|
| 目标业务 | 下单接口 `/api/order/create` | 明确单一目标，避免一次压多个链路 |
| 峰值 QPS 目标 | 3000 | 以活动峰值或容量目标为准 |
| 当前 P99 | 850ms | 线上基线值 |
| 目标 P99 | <= 300ms | 回归目标 |
| 允许错误率 | <= 0.1% | 业务可接受阈值 |
| 压测时长 | 20 分钟 | 建议包含预热阶段 |

## 二、容量估算模板

### 1) 关键公式

- 并发量估算：`并发数 = QPS x 平均响应时间(秒)`
- 理论实例数：`实例数 = 峰值QPS / 单机稳定QPS`
- 规划实例数：`规划实例数 = 理论实例数 x 冗余系数(1.2~1.5)`

### 2) 示例计算

- 目标峰值 QPS：3000
- 单机稳定 QPS：420
- 冗余系数：1.3

计算：

- 理论实例数 = `3000 / 420 = 7.14`，向上取整为 8
- 规划实例数 = `8 x 1.3 = 10.4`，向上取整为 11

结论：生产建议至少部署 11 个实例，并预留弹性扩容策略。

## 三、回归阈值模板（建议写进发布门禁）

| 维度 | 回归阈值 | 失败处理 |
|---|---|---|
| P95 延迟 | <= 180ms | 阻断发布，回滚到上一版本 |
| P99 延迟 | <= 300ms | 进入性能专项排查 |
| 错误率 | <= 0.1% | 启动降级策略并告警 |
| CPU 使用率 | <= 70% | 限流 + 扩容评估 |
| Full GC 频次 | <= 1 次/10 分钟 | 检查对象分配与堆参数 |

## 四、压测执行记录模板

| 时间 | 并发 | QPS | P99 | 错误率 | CPU | 结论 |
|---|---:|---:|---:|---:|---:|---|
| 10:00-10:05 | 500 | 1200 | 140ms | 0.00% | 35% | 稳定 |
| 10:05-10:10 | 1000 | 2100 | 210ms | 0.02% | 52% | 稳定 |
| 10:10-10:15 | 1500 | 2900 | 320ms | 0.15% | 76% | 超阈值 |
| 10:15-10:20 | 1200 | 2500 | 240ms | 0.03% | 61% | 调优后恢复 |

## 五、问题定位最小闭环

1. **发现异常**：指标超过阈值。
2. **确认范围**：单接口、单实例还是全链路。
3. **采集证据**：Trace、日志、GC、线程池、DB 慢查询。
4. **实施修复**：参数、代码、架构策略三选一或组合。
5. **回归验证**：重复压测，确认核心指标恢复。
6. **沉淀复盘**：形成可复用清单与阈值基线。

## 六、常见误区

- 只看平均延迟，不看 P99。
- 只压接口，不压下游依赖（DB/缓存/MQ）。
- 只看服务 CPU，不看连接池、线程池、GC。
- 压测通过后不写回归门禁，导致问题反复出现。

## Java 阈值判定示例（含注释，可直接运行）

**建议文件名：** `Main.java`  
**运行命令：** `javac Main.java && java Main`

```java
public class Main {
    public static void main(String[] args) {
        // 假设来自压测系统的当前指标
        double p99 = 320.0;      // ms
        double errorRate = 0.12; // %
        double cpu = 76.0;       // %

        boolean pass = p99 <= 300.0 && errorRate <= 0.1 && cpu <= 70.0;

        System.out.println("P99=" + p99 + "ms, 错误率=" + errorRate + "%, CPU=" + cpu + "%");
        if (pass) {
            System.out.println("结论: 通过回归阈值，可进入发布环节。");
        } else {
            System.out.println("结论: 未通过回归阈值，需阻断发布并执行排障。");
        }
    }
}
```

**预期输出示例：**

```text
P99=320.0ms, 错误率=0.12%, CPU=76.0%
结论: 未通过回归阈值，需阻断发布并执行排障。
```
