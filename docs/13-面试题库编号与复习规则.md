# 面试题库编号与复习规则

> 目标：统一题库管理，避免“题目很多但无法复习追踪”。

## 编号规范

格式：`[层级]-[优先级]-[序号]-[主题]`

示例：
- `L1-P0-01-HashMap线程安全`
- `L2-P1-06-缓存双写一致性`
- `L3-P2-03-分布式事务选型`

## 优先级定义

- `P0`：必问，必须能在 90 秒内完整口述
- `P1`：高频进阶，必须能讲场景和取舍
- `P2`：拉开差距，必须能讲方案演进和边界

## 题目记录模板

```markdown
### [L2-P1-06-缓存双写一致性]
- 一句话结论：
- 核心原理（3点）：
  1)
  2)
  3)
- 项目场景：
- 取舍与风险：
- 常见追问：
```

## 复习节奏

- **每日**：5 题快答（每题 60~90 秒）
- **每周**：20 题混合口述（按 P0->P1->P2）
- **每两周**：一次 45 分钟模拟面试

## 通过标准

- P0：回答完整率 >= 80%
- P1：可讲清“为什么这样做”
- P2：能明确方案边界和替代方案
